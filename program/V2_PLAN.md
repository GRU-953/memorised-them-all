## Media-skip strategy
"Add a config-driven skip filter at the TOP of convert_file (right after the not-a-file / 0-byte / oversize guards, before line 387 `ext = path.suffix.lower()`) that returns ConvResult(status='skipped', method='skipped-type', error=<category>) WITHOUT reading the file when the matching switch is on. Categories + extension sets: images _IMAGE_EXTS (existing), video NEW _VIDEO_EXTS {.mp4,.mov,.avi,.mkv,.webm,.m4v,.wmv,.flv}, audio _AUDIO_EXTS (existing) — all gated by skip_media (default ON for v2); fonts {.otf,.ttf,.ttc,.woff,.woff2} by skip_fonts; Google-Drive pointer files {.gdoc,.gsheet,.gform,.gdrive,.gslides,.gdraw} by skip_gdrive_pointers; junk {.tmp,.ds_store} + literal name '.DS_Store' by skip_junk. status='skipped' (not 'unsupported'/'failed') so digest stats count them honestly. The same filter is applied to archive-extracted members in _expand. This is the determinism enabler: with skip_media ON, the non-deterministic media converters — _try_whisper (faster-whisper/MLX, GPU, model download) and image OCR (Tesseract, not bit-stable across machines) — never run on the corpus path, so two digests of the same corpus produce byte-identical output. The skip switches are opt-out, so a future run that flips media on knowingly trades away the determinism guarantee."

## Risks
- Recall quality regresses to lexical-overlap grade: the 256-d md5 hashing embedding has no semantic similarity, so recall ranking + the embedding-confirmed entity merge in resolve.py lose accuracy vs neural embeddings. recall.py already degrades gracefully (lexical-overlap low_confidence, DOC-01) but this is a deliberate, large, user-visible tradeoff — document in CHANGELOG/ACCEPTANCE/RISKS.
- Hidden non-determinism beyond embeddings: community detection must use the deterministic path. graph.detect_communities prefers Leiden (igraph/leidenalg, seed=7) when the optional 'graph' extra is installed but falls to Louvain (seed=7) on the MIT core. Verify the path CI/users actually run (Louvain, no igraph) is byte-stable, since the 'graph' extra could introduce a different partition; the determinism test must run on the no-igraph path.
- On-disk vector incompatibility: pre-v2 projects may hold neural-dim vectors (e.g. 1024) in vectors.npz. recall.py:72-74 already guards via qv.shape != matrix.shape -> lexical fallback and store.load_vectors asserts no fixed dim, so recall degrades gracefully — but a re-digest (reset:true) is needed to get 256-d hash vectors. Note this in the migration section of CHANGELOG.
- Large cascading blast radius: deleting backends.py + lifecycle.py breaks imports in extract/digest/convert/embed/recall/server if any reference is missed. The plan removes call sites BEFORE deleting modules (steps 2-4 before 5-6); a final grep for `OllamaManager|backends|extract_mode|no_ollama` must return zero before the delete commits, or CI import-errors.
- memory_status response-shape break: dropping backend/ollama_running/ollama_inference/degraded/profile/idle_seconds is a public API change for an MCP/REST tool clients may consume (rest.py exposes memory_status). A v2.0.0 MAJOR is the right place but it must be documented in CHANGELOG/PUBLISH_MANIFEST, and any client recipe reading those keys reconciled.
- Removing open_mindmap from the MCP/REST tool surface is a tool-list contract change clients can call; needs a deprecation note in the v2.0.0 changelog, not a silent removal. A LIVE installed build keeps the tool until reinstall — source removal doesn't retroactively change a running instance.
- Recursive archive expansion is a zip-bomb/zip-quine amplifier and a Zip-Slip/tar-symlink-escape vector. The depth cap, cumulative-uncompressed budget, entry-count cap, and per-member path-traversal validation are load-bearing security controls — they MUST land with the expander, and the OOXML-vs-archive dispatch must key on extension (not zipfile.is_zipfile) or every Office doc gets wrongly unpacked.
- Temp-dir leak under timeout-kill: digest converts each file in a hard-killed subprocess (3s). Archive extraction must happen in the MAIN process under cfg.project_dir/_unpacked, not an mkdtemp inside the killable child, or a kill mid-extract leaks the scratch dir; cleanup goes in the digest-level teardown / _reset_project.
- scripts/check_versions.py gates version parity across 5 manifests on the tag-triggered publish train — the 2.0.0 bump must touch mta/__init__.py + manifest.json + server.json + plugin.json + marketplace.json in one commit or CI/release fails.
- Tesseract OCR + faster-whisper are kept as OPTIONAL converters but are NOT bit-deterministic across hardware; the determinism guarantee holds ONLY because skip_media defaults ON and removes them from the corpus path. If a future run flips media on, determinism silently breaks — flag prominently. MarkItDown itself isn't guaranteed byte-stable across versions either; MTA_AUTO_UPDATE should be off (or pinned) for a strict deterministic build.
- _scrub/_defang_fence existed mainly to sanitize LLM-emitted control tokens / second-order injection into summaries. With no LLM they are now only cheap hygiene on classical+converted text — keep them (still defends memory.md against control tokens in documents) but the rationale shifts; keep a minimal _scrub test.
