### v2.4.2 convergence — accepted / deferred (Med/Low, NOT release blockers)
| id | item (source) | sev | rationale / disposition |

| R-13 | Recall BM25 re-tokenizes the full recall-unit corpus + recomputes idf **every query**; recall-unit count uncapped (`recall._bm25_rank`, `digest._recall_units`) — perf cliff (~0.8s @50k units) | Med | **Resolved** (v2.6.0/WP-26): digest persists a deterministic pre-tokenised `bm25_index.json`; recall ranks from it (byte-identical results) → **~8.8× faster** @8k units. | R-14 | Entity resolution is O(n²) fuzzy+cosine up to a hard 1500-name cap that doubles as an unannounced quality cliff (`resolve.py`) | Med | **Resolved** (v2.6.0/WP-26): `(script, token-prefix)` blocking → 6.6% of O(n²) pairs (brute-force parity test proves same merges); dense n×n cosine dropped; silent 1500 cap → documented `MTA_RESOLVE_MAX_NAMES` (default 5000, 0=unbounded). WP-90 Bengali distinctness preserved. |
| R-15 | Recall loads the full unused `vectors.npz` matrix into RAM per query (BM25 never reads it) (`recall.py` via `load_vectors`) | Med | **Resolved** (v2.6.0/WP-26): `store.load_meta` reads only `vectors.json`; the matrix is never loaded on the recall path (no_memory/torn semantics preserved). |
| R-16 | `resolve._norm` skeleton-merged **non-Bengali Indic** (Devanagari/Tamil); only the Bengali block was preserved | Low | **Resolved** (v2.7/WP-101): `_norm`'s combining-mark carve-out and `_NORM_RE` now keep the **whole Brahmic range** (U+0900–U+0DFF, Devanagari…Sinhala), so काली≠कुल / கடல்≠கடா no longer collapse to consonant skeletons. Preserving more marks only ever *reduces* merges (safe over-split); the per-script tightening + minimal-pair proofs are WP-121. Pinned by `tests/test_resolve.py::test_norm_preserves_non_bengali_indic_distinctness`. |
| R-17 | Fuzzy merge could collapse near-duplicate short Bengali names at the 88 ratio (e.g. করিম/করিমা) (`resolve.py` fuzz_threshold) | Low | **Resolved** (v2.7/WP-102): `_eff_threshold` raises the required `token_set_ratio` for short names (≤4 chars → 95, ≤6 → 91) so করিম/করিমা (≈89) no longer merge, while long-name typos still merge at the base 88. Monotone/deterministic; only ever *raises* the bar (safe over-split). Pinned by `test_eff_threshold_*` + `test_short_names_do_not_overmerge_*`. |
